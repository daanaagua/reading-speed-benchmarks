# Worked Examples

Quick examples using the formula:

`reading_time_minutes = word_count / wpm`

## Example 1: Fiction chapter

- Word count: 3,200
- WPM: 260 (mystery-thriller baseline)
- Time: 12.3 minutes

## Example 2: Academic paper

- Word count: 6,000
- WPM: 100 (STEM baseline)
- Time: 60 minutes

## Example 3: Legal review

- Word count: 12,000
- WPM: 70 (legal baseline)
- Time: 171.4 minutes (~2h 51m)

## Practical workflow

1. Start with genre benchmark defaults.
2. Run a personal reading speed test.
3. Save your own WPM profile.
4. Recalculate time estimates with personal values.
